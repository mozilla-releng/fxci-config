## Expected Gaps (16 resources)

These resources exist in community-tc but are intentionally NOT generated by fxci-config:

### Worker-pool roles (14)
`Role=worker-pool:proj-fuzzing/*` — Community-tc's `generate/workers.py` creates a Role for each worker pool granting `auth:sentry:generic-worker`. fxci's worker pool generator does not create these roles. If needed, they can be added via grants.yml.

### Secret (1)
`Secret=worker-pool:proj-fuzzing/bugmon-pernosco` — Community-tc creates a secret for docker-worker pools containing DNS configuration. fxci handles this differently.

### Externally-managed hook role (1)
`Role=hook-id:project-fuzzing/failed-log-ingestor` — This hook is referenced in grants but not defined in `fuzzing.yml`'s hooks section. It's part of the externally-managed fuzzing-tc-config deployment.

## How to Reproduce

```bash
# Generate community-tc baseline
cd ~/git/community-tc-config
source .venv/bin/activate
tc-admin generate --without-secrets --json > /tmp/community-tc-all.json

# Generate fxci baseline (before changes)
cd ~/git/fxci-config
source .venv/bin/activate
git stash  # stash fuzzing changes
python generate_subset.py --environment=firefoxci --json > /tmp/fxci-all-before.json
git stash pop

# Generate fxci with fuzzing changes
python generate_subset.py --environment=firefoxci --json > /tmp/fxci-all-after.json

# Compare
python3 tests/test_fuzzing_migration.py \
    --community-tc-json /tmp/community-tc-all.json \
    --fxci-before-json /tmp/fxci-all-before.json \
    --fxci-after-json /tmp/fxci-all-after.json
```
